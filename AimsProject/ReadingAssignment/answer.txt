Answer:
	- Lớp cha "Media" nên là lớp implement giao diện Comparable. Cụ thể là: public abstract class Media implements Comparable<Media>. Vì giỏ hàng (Cart) của chúng ta chứa 
danh sách các đối tượng Media (ArrayList<Media>). Khi gọi Collections.sort(), Java cần biết cách so sánh các phần tử kiểu Media với nhau. Các lớp con (Book, DigitalVideoDisc, CompactDisc)
sẽ tự động kế thừa khả năng so sánh này từ Media.
	- Trong lớp Media, ta sẽ ghi đè (override) phương thức compareTo(Media other). Giả sử ta chọn thứ tự mặc định là "Theo Tiêu đề (A-Z) rồi đến Giá (Giảm dần)". Đoạn code cụ thể sẽ như sau:
	@Override
	public int compareTo(Media other) {
	// Ưu tiên 1: So sánh theo Title (Alphabetical)
	int titleCompare = this.getTitle().compareToIgnoreCase(other.getTitle());
	if (titleCompare != 0) {
	return titleCompare;
	}
	// Ưu tiên 2: Nếu Title giống nhau, so sánh theo Cost (Giảm dần)
	return Float.compare(other.getCost(), this.getCost());
	}
	- Không. Giao diện Comparable được dùng để định nghĩa "Natural Ordering" (Thứ tự sắp xếp tự nhiên/mặc định) của một đối tượng. Mỗi class chỉ có thể có duy nhất 1 hàm  hàm compareTo(). 
Do đó, bạn chỉ có thể chọn 1 trong 2 quy tắc để làm mặc định.
	- Lợi dụng tính Đa hình (Polymorphism) bằng cách ghi đè (Override) lại hàm compareTo() ngay bên trong lớp con DigitalVideoDisc. Ý tưởng:
	+ Nếu đối tượng đem ra so sánh cũng là một chiếc đĩa DVD -> Áp dụng luật riêng của DVD (Title -> Length -> Cost).
	+ Nếu đối tượng đem ra so sánh là Sách hoặc CD (không phải DVD) -> Đẩy ngược lên cho lớp cha (Media) giải quyết bằng lệnh super.compareTo().
