Question:
- If you create a constructor method to build a DVD by title then create a constructor method to build 
a DVD by category. Does JAVA allow you to do this?
Answer:
Không. Trong Java, quy tắc Nạp chồng phương thức (Overloading) phân biệt các hàm dựa trên kiểu dữ liệu
và số lượng tham số chứ hoàn toàn không quan tâm đến tên biến. Vì cả title và category đều là kiểu chuỗi 
(String), hai hàm tạo này sẽ bị trùng lặp vì có chữ ký hàm giống hệt nhau là DigitalVideoDisc(String). 
Nếu bạn gõ lệnh tạo đĩa và truyền vào một chuỗi chữ, hệ thống sẽ báo lỗi ngay lập tức vì nó không thể tự 
phân biệt được bạn đang muốn lưu chuỗi đó làm tiêu đề hay thể loại.
Question: Is JAVA a Pass by Value or a Pass by Reference programming language?
Answer:
Java là ngôn ngữ Pass-by-Value (truyền tham trị) trong mọi trường hợp. Khi bạn truyền một đối tượng vào 
phương thức, thực chất Java đang tạo ra một bản sao của giá trị tham chiếu (địa chỉ vùng nhớ) và truyền 
bản sao đó đi. Điều này có nghĩa là bạn có thể thay đổi nội dung bên trong đối tượng mà tham chiếu đó trỏ 
tới, nhưng bạn tuyệt đối không thể thay đổi chính biến gốc để nó trỏ sang một đối tượng hoàn toàn khác.
